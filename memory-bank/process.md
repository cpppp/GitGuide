## V3.2 计划：AI问答功能升级（源码级别分析）

**优先级**：高\
**预计工作量**：7 天\
**目标**：实现源码级别的AI问答，支持对特定文件（如main.py）进行深度分析\
**依赖**：V3.1 文档质量提升完成 ✅

### 核心特性目标

| 特性 | 描述 |
|:---|:---|
| **源码级别分析** | AI能够读取并分析仓库中的任何文件内容，支持对特定文件提问 |
| **文件路径引用** | 支持用户指定文件，如"分析这个项目的main.py文件" |
| **上下文感知** | AI回答时能够引用具体的代码行、函数定义、import语句 |
| **多轮对话** | 支持追问和上下文关联的连续对话 |

## V3.2 实施步骤

### 步骤 1：知识库增强 - 源码索引构建

**优先级**：最高\
**预计时间**：2 天

**目标**：构建包含完整源码的知识库，支持按文件路径精确检索

**任务**：

- [x] 实现 `agents/chat/source_code_indexer.py`
  - 扫描仓库所有源文件（支持语言：Python, JavaScript, TypeScript, Java, Go, Rust, C++）
  - 提取文件内容、函数定义、类定义、import/export 语句
  - 构建文件路径索引，支持按路径检索
  - 限制索引文件大小（单文件最大 100KB）

- [x] 更新 `agents/chat/knowledge_builder.py`
  - 集成 SourceCodeIndexer
  - 创建文件级别的向量嵌入
  - 支持按文件路径过滤检索结果

- [x] 实现 Chroma 向量数据库 schema 更新
  - 添加 `file_path` 字段索引
  - 添加 `line_range` 字段（支持行号范围检索）
  - 添加 `code_language` 字段

**验证方法**：

- [x] 能索引任意GitHub仓库的源码文件
- [x] 能按文件路径精确检索到指定文件
- [x] 检索响应时间 < 500ms

***

### 步骤 2：RAG检索增强 - 文件级别检索

**优先级**：最高\
**预计时间**：2 天

**目标**：增强RAG检索，支持用户指定文件或按文件路径筛选

**任务**：

- [x] 更新 `agents/chat/rag_retriever.py`
  - 实现 `retrieve_by_file_path(file_path)` 方法
  - 实现 `retrieve_code_context(query, max_lines=50)` 方法（返回代码上下文）

- [x] 实现 Prompt 模板优化
  - 设计专门的代码分析Prompt模板
  - 支持引用具体文件和行号
  - 包含代码解释的标准格式

**验证方法**：

- [x] 用户问"分析main.py"，AI能返回main.py的完整内容摘要
