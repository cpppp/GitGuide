## 1. 产品概述

**GitGuide** 是一个帮助开发者快速上手任意 GitHub 仓库的智能化工具。用户只需输入 GitHub 仓库 URL，系统即可自动分析仓库内容，生成一份定制的学习文档 + 启动指南，并结合 AI 能力提供交互式答疑，让用户可以边学边问，快速理解项目结构并成功在本地运行。

**核心价值主张**：让新学习者在 5 分钟内理解项目，30 分钟内运行起来。

## 2. 目标用户

### 2.1 主要用户画像

| 用户类型 | 典型场景 | 核心需求 |
|:---|:---|:---|
| **开源项目新手** | 刚接触某个开源项目，希望快速了解 | 快速入门、核心概念、最小化运行 |
| **代码接手者** | 需要接手同事或他人代码 | 架构理解、模块关系、运行指南 |
| **编程学习者** | 通过实际项目练习但不知从何入手 | 学习路径、代码解释、实践指导 |
| **技术评估者** | 快速评估开源项目可行性 | 项目概览、技术栈、同类对比 |
| **文档编写者** | 需要生成项目文档基础框架 | 文档模板、结构化输出 |

### 2.2 新学习者核心痛点

| 痛点 | 描述 | 严重程度 | GitGuide 解决方案 |
|:---|:---|:---|:---|
| **不知道从哪里开始** | 项目太大，找不到入口，不知道先看什么 | ⭐⭐⭐⭐⭐ | 快速入门文档 + 学习路径 |
| **不知道项目能做什么** | 不理解核心功能和价值，不知道是否满足需求 | ⭐⭐⭐⭐⭐ | 项目概览文档 + 功能列表 |
| **不知道怎么运行起来** | 环境配置复杂，依赖多，版本冲突 | ⭐⭐⭐⭐⭐ | 安装部署文档 + 一键命令 |
| **遇到问题不知道怎么解决** | 报错看不懂，不知道怎么调试，找不到解决方案 | ⭐⭐⭐⭐ | 故障排查文档 + AI 问答 |
| **不知道代码结构** | 文件太多，不知道哪个是核心，不知道模块关系 | ⭐⭐⭐⭐ | 架构设计文档 + 代码图谱 |
| **不知道怎么扩展** | 想基于项目开发，不知道从哪里改，不知道接口 | ⭐⭐⭐ | 开发指南文档 + API 参考 |

### 4.4 AI 问答功能设计（核心）

#### 4.4.1 设计目标

AI 问答功能必须**真正理解仓库**，能够回答新学习者的实际问题：

| 用户问题类型 | 示例问题 | 期望的回答质量 |
|:---|:---|:---|
| **项目理解** | "这个项目是做什么的？" | 基于项目类型、README、功能列表的准确描述 |
| **运行指导** | "怎么运行起来？" | 基于入口点、安装指南、依赖配置的具体步骤 |
| **技术栈** | "用了哪些技术？" | 基于依赖关系、框架检测的准确列表 |
| **代码理解** | "这个函数是做什么的？" | 基于代码片段、注释文档的详细解释 |
| **问题排查** | "报错了怎么办？" | 基于故障排查文档、常见错误的解决方案 |
| **扩展开发** | "怎么添加新功能？" | 基于架构模式、API接口的开发指导 |

## 7. 用户流程

### 7.1 基础流程（MVP）

1. 用户访问 GitGuide 首页
2. 在输入框中粘贴 GitHub 仓库 URL，点击"生成"按钮
3. 系统展示加载状态，后台抓取仓库并进行分析
4. 分析完成后，页面跳转至结果页，展示学习文档和启动指南
5. 用户可阅读文档，并随时向 AI 提问

## 11. Multi-Agent架构设计

### 11.1 架构图

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Orchestrator (入口层)                           │
│                                                                             │
│   职责：克隆仓库 → 获取仓库信息 → 调用 Workflow → 清理临时目录 → 进度回调      │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Workflow (核心编排层)                           │
│                                                                             │
│   WorkflowState 状态管理 + 任务规划 + 并行调度 + 质量审核 + 结果整合          │
│                                                                             │
│   工作流程：初始化 → 分析(并行) → 生成(并行) → 审核 → 输出                     │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
          ┌───────────────────────────┼───────────────────────────┐
          ▼                           ▼                           ▼
┌───────────────────┐       ┌───────────────────┐       ┌───────────────────┐
│   Analyzer Team   │       │   Generator Team  │       │     Chat Agent    │
│    (并行执行)      │       │    (并行执行)      │       │    (V3.2新增)    │
├───────────────────┤       ├───────────────────┤       ├───────────────────┤
│ TypeAnalyzer      │       │ QuickStartGen     │       │ KnowledgeBuilder  │
│ StructureAnalyzer │       │ OverviewGen       │       │ RAGRetriever      │
│ DependencyAnalyzer│       │ ArchitectureGen   │       │ ConversationMgr   │
│ CodePatternAnalyzer│      │ InstallGuideGen   │       │                   │
│                   │       │ TutorialGen       │       │                   │
│                   │       │ DevGuideGen       │       │                   │
│                   │       │ TroubleshootGen   │       │                   │
└───────────────────┘       └───────────────────┘       └───────────────────┘
```

### 11.2 Agent 职责说明

| Agent 类型 | 组件 | 职责 |
|:---|:---|:---|
| **编排层** | Orchestrator | 入口管理、仓库克隆、进度回调、资源清理 |
| | Workflow | 状态管理、并行调度、质量审核、结果整合 |
| **Analyzer Team** | TypeAnalyzer | 项目类型识别、语言/框架/构建系统检测 |
| | StructureAnalyzer | 目录结构分析、模块识别、入口点提取 |
| | DependencyAnalyzer | 依赖关系分析、版本兼容性检查 |
| | CodePatternAnalyzer | 代码模式识别、架构风格分析 |
| **Generator Team** | QuickStartGen | 快速入门文档生成 |
| | OverviewGen | 项目概览文档生成 |
| | ArchitectureGen | 架构设计文档生成 |
| | InstallGuideGen | 安装部署文档生成 |
| | TutorialGen | 使用教程文档生成 |
| | DevGuideGen | 开发指南文档生成 |
| | TroubleshootGen | 故障排查文档生成 |
| **Chat Agent** | KnowledgeBuilder | 构建仓库知识库 |
| | RAGRetriever | 检索增强生成 |
| | ConversationManager | 多轮对话管理 |

### 11.3 架构设计原则

| 原则 | 说明 |
|:---|:---|
| **简洁高效** | 移除冗余的 Supervisor 层，职责合并到 Workflow |
| **并行优先** | Analysis 和 Generation 阶段采用并行执行 |
| **状态驱动** | WorkflowState 实现 Agent 间的数据传递 |
| **质量可控** | 内置质量审核机制，确保文档质量 |
