## 1. 产品概述

**GitGuide** 是一个帮助开发者快速上手任意 GitHub 仓库的智能化工具。用户只需输入 GitHub 仓库 URL，系统即可自动分析仓库内容，生成一份定制的学习文档 + 启动指南，并结合 AI 能力提供交互式答疑，让用户可以边学边问，快速理解项目结构并成功在本地运行。

***

## 2. 目标用户

- 刚接触某个开源项目，希望快速了解的开发者。
- 需要接手同事或他人代码，希望快速熟悉项目结构的人。
- 学习编程，想通过实际项目练习但不知从何入手的学员。
- 技术团队负责人，需要快速评估开源项目可行性。
- 技术文档编写者，需要生成项目文档基础框架。

## 4. 用户流程

### 4.1 基础流程（MVP）

1. 用户访问 GitGuide 首页。
2. 在输入框中粘贴 GitHub 仓库 URL，点击"生成"按钮。
3. 系统展示加载状态，后台抓取仓库并进行分析。
4. 分析完成后，页面跳转至结果页，展示：
   - 左侧：学习文档（分章节）
   - 右侧：启动指南（步骤列表）
   - 底部或侧边：AI 聊天窗口
5. 用户可阅读文档，并随时向 AI 提问，AI 回答。

### 4.2 增强流程（迭代后）

1. **输入阶段**：
   - URL 格式实时验证
   - 历史记录快速选择
   - 收藏仓库快速访问
2. **分析阶段**：
   - 实时进度条显示
   - 分步骤状态提示
   - 预计剩余时间
   - 支持取消任务
3. **结果展示**：
   - 学习文档（分章节）
   - 启动指南（步骤列表）
   - 代码图谱可视化
   - AI 聊天窗口
4. **后续操作**：
   - 导出为多种格式
   - 分享分析结果
   - 收藏仓库
   - 提供反馈

### 4.3 数据持久化流程（v3.0新增）

1. **AI问答记录**：
   - 用户提问自动保存到数据库
   - 切换仓库时清空当前问答记录
   - 支持查看历史问答记录
   - 支持删除单条或清空全部记录
2. **仓库文档存储**：
   - 分析完成后自动保存文档到数据库
   - 重新分析时更新已有文档
   - 支持离线查看已分析仓库
   - 支持删除已保存的文档

## 7. 数据库设计（v3.0新增）

### 7.1 核心数据表

```sql
-- 仓库分析记录表
CREATE TABLE repositories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url VARCHAR(500) NOT NULL UNIQUE,
    name VARCHAR(200),
    description TEXT,
    language VARCHAR(50),
    stars INTEGER DEFAULT 0,
    learning_doc TEXT,
    setup_guide TEXT,
    analysis_result TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- AI问答记录表
CREATE TABLE chat_messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id INTEGER REFERENCES repositories(id),
    role VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户收藏表
CREATE TABLE favorites (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id INTEGER REFERENCES repositories(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 分析历史表
CREATE TABLE analysis_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id INTEGER REFERENCES repositories(id),
    analyzed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 7.2 数据关系

```
┌─────────────────┐     ┌─────────────────┐
│  repositories   │     │  chat_messages  │
├─────────────────┤     ├─────────────────┤
│ id (PK)         │◄────│ repo_id (FK)    │
│ url             │     │ role            │
│ name            │     │ content         │
│ learning_doc    │     │ created_at      │
│ setup_guide     │     └─────────────────┘
│ ...             │
└────────┬────────┘
         │
         │     ┌─────────────────┐
         │     │    favorites    │
         │     ├─────────────────┤
         └────►│ repo_id (FK)    │
               │ created_at      │
               └─────────────────┘
```

## 8. Multi-Agent架构升级（v3.1）

### 8.1 新架构设计

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           Supervisor Agent                               │
│                    (任务规划、进度协调、质量控制)                           │
│                                                                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐   │
│  │   Planner   │  │  Scheduler  │  │  Reviewer   │  │  Optimizer  │   │
│  │  (规划器)   │  │  (调度器)   │  │  (审核器)   │  │  (优化器)   │   │
│  └─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘   │
└───────────────────────────────┬─────────────────────────────────────────┘
                                │
        ┌───────────────────────┼───────────────────────┐
        │                       │                       │
        ▼                       ▼                       ▼
┌───────────────────┐   ┌───────────────────┐   ┌───────────────────┐
│   Analyzer Team   │   │   Generator Team  │   │   Research Team   │
│    (分析团队)      │   │    (生成团队)      │   │    (调研团队)      │
├───────────────────┤   ├───────────────────┤   ├───────────────────┤
│ TypeAnalyzer      │   │ ReadmeGenerator   │   │ CodeAnalyzer      │
│ StructureAnalyzer │   │ LearningDocGen    │   │ APIAnalyzer       │
```
