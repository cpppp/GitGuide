## 1. 产品概述

**GitGuide** 是一个帮助开发者快速上手任意 GitHub 仓库的智能化工具。用户只需输入 GitHub 仓库 URL，系统即可自动分析仓库内容，生成一份定制的学习文档 + 启动指南，并结合 AI 能力提供交互式答疑，让用户可以边学边问，快速理解项目结构并成功在本地运行。

***

## 2. 目标用户

- 刚接触某个开源项目，希望快速了解的开发者。
- 需要接手同事或他人代码，希望快速熟悉项目结构的人。
- 学习编程，想通过实际项目练习但不知从何入手的学员。
- 技术团队负责人，需要快速评估开源项目可行性。
- 技术文档编写者，需要生成项目文档基础框架。

## 4. 新学习者需求分析

### 4.1 核心痛点

对于一个新接触项目的学习者，面临的核心问题：

| 痛点              | 描述                    | 严重程度  |
| :-------------- | :-------------------- | :---- |
| **不知道从哪里开始**    | 项目太大，找不到入口，不知道先看什么    | ⭐⭐⭐⭐⭐ |
| **不知道项目能做什么**   | 不理解核心功能和价值，不知道是否满足需求  | ⭐⭐⭐⭐⭐ |
| **不知道怎么运行起来**   | 环境配置复杂，依赖多，版本冲突       | ⭐⭐⭐⭐⭐ |
| **遇到问题不知道怎么解决** | 报错看不懂，不知道怎么调试，找不到解决方案 | ⭐⭐⭐⭐  |
| **不知道代码结构**     | 文件太多，不知道哪个是核心，不知道模块关系 | ⭐⭐⭐⭐  |
| **不知道怎么扩展**     | 想基于项目开发，不知道从哪里改，不知道接口 | ⭐⭐⭐   |

### 4.2 需要的文档类型

基于痛点分析，新学习者需要的文档类型：

| 文档类型      | 核心目的    | 关键内容                      | 优先级 |
| :-------- | :------ | :------------------------ | :-- |
| **快速入门**  | 5分钟了解项目 | 一句话概括、核心概念图解、最小化运行命令      | 高   |
| **项目概览**  | 理解项目全貌  | 背景、功能列表、技术选型、适用场景、同类对比    | 高   |
| **架构设计**  | 理解系统结构  | 架构图、模块说明、数据流、设计决策、技术债务    | 高   |
| **安装部署**  | 让项目跑起来  | 环境要求、安装步骤、配置说明、常见问题、验证方法  | 高   |
| **使用教程**  | 学会使用功能  | 基础用法、进阶用法、API参考、最佳实践、示例代码 | 中   |
| **开发指南**  | 参与项目开发  | 目录结构、代码规范、开发环境、测试指南、贡献流程  | 中   |
| **故障排查**  | 解决遇到的问题 | 常见错误、调试技巧、日志说明、性能排查、FAQ   | 中   |
| **API文档** | 接口调用参考  | 端点列表、请求示例、响应格式、错误码、认证方式   | 低   |

### 4.3 辅助素材类型

| 素材类型       | 内容                    | 作用     |
| :--------- | :-------------------- | :----- |
| **可视化素材**  | 架构图、目录树、流程图、时序图、代码图谱  | 降低理解门槛 |
| **代码素材**   | 核心代码片段、配置示例、使用示例、测试用例 | 可复制使用  |
| **交互式素材**  | AI问答、代码导航、实时演示        | 即时解惑   |
| **学习路径素材** | 学习路线图、知识点清单、练习任务      | 系统学习   |

**核心目标**：升级 Multi-Agent 架构，实现并行处理，提高分析效率

| **Supervisor Agent** | 实现 Planner、Scheduler、Reviewer、Optimizer 四大核心组件                           | 高   |

| **并行执行引擎**           | 使用 LangGraph + asyncio 实现多Agent并行执行                                      | 高   |
| **共享上下文机制**          | Agent间状态共享，避免信息丢失                                                        | 中   |

## 7. 用户流程

### 7.1 基础流程（MVP）

1. 用户访问 GitGuide 首页。
2. 在输入框中粘贴 GitHub 仓库 URL，点击"生成"按钮。
3. 系统展示加载状态，后台抓取仓库并进行分析。
4. 分析完成后，页面跳转至结果页，展示：
   - 左侧：学习文档（分章节）
   - 右侧：启动指南（步骤列表）
   - 底部或侧边：AI 聊天窗口
5. 用户可阅读文档，并随时向 AI 提问，AI 回答。

### 7.2 增强流程（V3.2后）

1. **输入阶段**：
   - URL 格式实时验证
   - 历史记录快速选择
   - 收藏仓库快速访问
2. **分析阶段**：
   - 实时进度条显示
   - 分步骤状态提示（Planning → Analysis → Generation → Review → Optimization）
   - 预计剩余时间
   - 支持取消任务
3. **结果展示**：
   - 文档标签页切换（快速入门、项目概览、架构设计、安装部署、使用教程、开发指南、故障排查）
   - 辅助素材展示（架构图、目录树、代码图谱、示例代码）
   - 文档质量评分展示
   - AI 聊天窗口
4. **后续操作**：
   - 导出为多种格式
   - 分享分析结果
   - 收藏仓库

## 10. 数据库设计

### 10.1 核心数据表

```sql
-- 仓库分析记录表
CREATE TABLE repositories (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    url VARCHAR(500) NOT NULL UNIQUE,
    name VARCHAR(200),
    description TEXT,
    language VARCHAR(50),
    stars INTEGER DEFAULT 0,
    learning_doc TEXT,
    setup_guide TEXT,
    quick_start TEXT,
    architecture_doc TEXT,
    usage_tutorial TEXT,
    dev_guide TEXT,
    troubleshooting TEXT,
    analysis_result TEXT,
    quality_score INTEGER DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- AI问答记录表
CREATE TABLE chat_messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id INTEGER REFERENCES repositories(id),
    role VARCHAR(20) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 用户收藏表
CREATE TABLE favorites (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    repo_id INTEGER REFERENCES repositories(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
